Injection SQL

💥 Qu’est-ce qu’une injection SQL ?

Une injection SQL est une attaque qui exploite une faille dans les formulaires web ou les requêtes SQL mal sécurisées, permettant à un attaquant d’exécuter du code malveillant dans la base de données.

🧪 Comment ça fonctionne ?

Un champ de saisie (ex: identifiant) est manipulé pour injecter une commande SQL (comme ' OR 1=1 --) dans une requête. L'attaquant peut alors contourner l'authentification ou accéder aux données sans autorisation.

📌 Exemples d’exploitation

  • Connexion sans mot de passe grâce à ' OR 1=1 --
  • Récupération de toutes les données utilisateur
  • Suppression ou modification des tables de la base

⚠️ Risques

  • Vol massif de données sensibles
  • Destruction ou corruption de base de données
  • Prise de contrôle totale de l’application

🛡️ Comment s’en protéger ?

  • Utiliser des requêtes préparées (requêtes paramétrées)
  • Échapper correctement les caractères spéciaux
  • Limiter les droits SQL (principes du moindre privilège)
  • Ne jamais faire confiance aux données saisies par l’utilisateur
